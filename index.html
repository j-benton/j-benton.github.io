Worship Setlist Generator

Capstone Project for General Assembly Data Science Immersive

For this project, I created a locally hosted Flask application that recommended songs for church services based on user input. I used:

  • Python
  • Requests
  • BeautifulSoup4
  • CountVectorizer
  • spaCy
  • VaderSentiment
  • Flask
  • HTML
  • CSS
All code was written in Python, except for the portion of my Flask app that used HTML and CSS. Using Requests and BeautifulSoup4, I wrote code that scraped praisecharts.com and biblestudytools.com for song metadata/lyrics and biblical text, respectively. I then used the spaCy NLP library to tokenize, lemmatize, and vectorize all text data. Finally, I defined a function that took in four types of user input: Bible verses (e.g. "John 3:16"), sermon themes/keywords (e.g. "forgiveness"), set length (e.g. "5" songs), and the ratio of upbeat/slower songs. My function returned a setlist to user specifications with the songs containing lyrics most similar to the given Bible verses and sermon themes/keywords.

This project was inspired by my experience as a hired drummer for various churches and seeing the constant need for a way to efficiently construct worship setlists that supported and enhanced a given service's message.

COVID-19 Twitter Sentiment Analysis

Client/Group Project for General Assembly Data Science Immersive

For this group project, my team analyzed the sentiments of tweets from across the U.S. to see if states' differing responses to COVID-19 affected Twitter users' attitudes. We used:

  • Python
  • GetOldTweets3
  • spaCy
  • CountVectorizer
  • VaderSentiment
  • Matplotlib
We worked with GetOldTweets3 to get tweets from key cities in states with the most severe and least severe lockdown measures during most of April 2020. As a control, we also pulled tweets from the same cities and dates from 2019. We ended up with more than 700,000 tweets from twenty cities. We then used spaCy to process the text data and VaderSentiment for sentiment analyziz. Finally, we plotted the sentiments to identify trends and connections. However, tweet sentiments didn't seem to be affected by differences in statewide policies responding to COVID-19, nor did sentiments change much between 2019 and 2020.

The West Wing vs. House of Cards

NLP Project for General Assembly Data Science Immersive

For this project, I used NLP and Reddit's Pushshift API to predict subreddit origin for posts from two subreddits: The West Wing and House of Cards. I used:

  • Python
  • Requests
  • BeautifulSoup4
  • NLTK
  • GridsearchCV
  • Pipeline
  • CountVectorizer
  • TfidfVectorizer
  • Logistic Regression
  • Multinomial Naive Bayes
First I gathered 4000 posts from each subreddit, pulling 2000 both before and after Trump's election in 2016 for extra analysis (I wanted to predict not only which subreddit a post came from but also whether it was posted pre- or post-Trump). I then processed the data using the NLTK data, including tokenization, lemmatization, and removing stopwords. I then used both CountVectorizer and TfidfVectorizer with Logistic Regression, KNN, Multinomial Naive Bayes, and Binomial Naive Bayes models to see which performed most accurately. Logistic Regression (w/TfidfVec) and Multinomial NB (w/CountVec) were chosen.

I then trained and tested my two models on my processed text data and found that they predicted a post's subreddit origin with fairly high accuracy (~80%). However, I had intentionally left in dead-giveaway words like the actors'/characters' names. After removing those, accuracy dropped down to about 65% (average between the two), which was still above the baseline of 50%. My Trump analysis was unfruitful, with my models performing as well as the baseline.
